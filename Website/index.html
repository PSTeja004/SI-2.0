S-Interface 2.0

     About S-Interface 2.0

     
      The S-Interface, previously known as OTACS, is a modular system built in C# and serves as a communal software development project.
      The platform allows contributions from anyone, enabling the addition of modules with specific functionality referred to as plug-ins.
      These plug-ins play a crucial role in extending the capabilities of the S-Interface and must adhere to programming guidelines outlined.
      In its current plug-in library, the S-Interface features three sensor acquisition channels and four algorithms.

      Functionally speaking, the plug-ins are of two major varieties:
      (a) Sensor channels
      (b) Algorithm implementations

      We aim to develop a prototype of S-Interface 2.0 by addressing the following main requirements:
      S-Interface 1.0 was developed using .NET framework (Winforms) and C#.
      The main challenge with this software product is that it can only be installed and used in Windows platform.
      Also, Microsoft has discontinued Winforms so it has been difficult to upgrade and maintain. So, we need to rewrite the software using more platform independent language such as Python.

      CPL has developed several intensive legacy algorithms in C# in order to be used as plugins in S-Interface 1.0.
      It will not be possible to rewrite all these algorithms in Python at this time.
      So, we need to investigate if there are any C# to Python wrappers available that will make it feasible to use existing SInterface 1.0 plugins in S-Interface 2.0.
